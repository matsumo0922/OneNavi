## 1. TTSデータベースの概要

### ファイル

| パス (APK内) | サイズ | 言語 |
|---|---|---|
| `assets/ja-JP/car/ttsstatic.db` | 約8MB | 日本語 |
| `assets/en-US/car/ttsstatic.db` | 約6MB | 英語 |

形式は SQLite 3。デコンパイル後は `drive-supporter-native/assets/` に展開される。

### テーブル構造

#### `tts_cache_t` (音声フレーズ本体)

```sql
CREATE TABLE tts_cache_t (
    id              INTEGER PRIMARY KEY,
    sys_ver         INTEGER,
    engine          TEXT,
    speaker         INTEGER,
    speaker_rev     INTEGER,
    type            INTEGER,       -- 0: 通常テキスト
    text            TEXT NOT NULL,  -- フレーズ本文
    volume          INTEGER,
    speed           INTEGER,
    pitch           INTEGER,
    depth           INTEGER,
    latitude        INTEGER,
    longitude       INTEGER,
    data            BLOB,          -- 合成済み音声データ (OGG)
    codec           TEXT NOT NULL,
    timestamp       INTEGER,
    UNIQUE(engine, speaker, type, text, volume, speed, pitch, depth, latitude, longitude)
);
```

## 2. フレーズ組み立てアーキテクチャ

### パイプライン全体

```
GPS位置更新
  |
  v
状態マシン (d.java: ターン案内 / c.java: 道なり案内)
  |  メソッド f(NTGuidanceRouteMatchResult, String) が毎回呼ばれる
  v
NTNvGuidanceManager (JNI)
  |  createGuidePhrase()         -- 通常の方向案内
  |  createFixedGuidePhrase()    -- 固定フレーズ (開始/終了/リルート等)
  |  createArrivalGuidePhrase()  -- 到着時
  |  createGuidePhraseAtIndex()  -- 特定地点の案内
  v
PhraseData[] (ネイティブエンジンが返すフレーズ断片の配列)
  |
  v
Java側で後処理 (NTPlayMediaGuideTts.m())
  |  - 目的地名の差し替え
  |  - 経由地名の差し替え
  |  - 終了フレーズの差し替え
  v
TTS合成 or 静的音声キャッシュ参照
  |  ttsstatic.db → キャッシュ済み .ogg
  |  未キャッシュ → オンラインTTS合成 → ttsdynamic.db に保存
  v
順番に再生
```

### ネイティブライブラリ

`NTNvGuidanceManager` は以下のネイティブライブラリ群をロードする:

```
libsqlite3.so / libTIAccess.so / libRS6.so
libGuidanceEngine.so / libNvSearcher.so
libNvGuidanceEngine.so / libGuidanceManager.so
```

フレーズの組み立て (どの断片をどの順番で並べるか) は **ネイティブ側で完結** する。
Java側が受け取るのは、組み立て済みの `NTTtsPhraseData[]` 配列。

### PhraseData の構造

**TTSモード (`NTPlayMediaGuideTts.PhraseData`):**

| フィールド | 型 | 説明 |
|---|---|---|
| `priority` | int | 再生優先度 (0-4) |
| `vibPattern` | int | バイブレーションパターン (0-3) |
| `phraseTypes` | int[] | 案内種別コード |
| `keys` | int[] | 音声ファイルID (オフラインモード用) |
| `onlineTtsPhraseData` | NTTtsPhraseData[] | オンラインTTS用フレーズ配列 |
| `offlineTtsPhraseData` | NTTtsPhraseData[] | オフラインTTS用フレーズ配列 (フォールバック) |

**NTTtsPhraseData:**

| フィールド | 型 | 説明 |
|---|---|---|
| `mPhrase` | String | フレーズ本文 (例: "およそ300m先") |
| `mType` | int | 合成タイプ (0=プレーンテキスト, 1=タグ付き, 2=X-SAMPA, 3=音声記号, 100=効果音) |
| `mVolume` | int | 音量 (MIN_VALUE=デフォルト) |
| `mSpeed` | int | 速度 |
| `mPitch` | int | ピッチ |
| `mDepth` | int | 深さ |
| `mLatitude` | int | 緯度 (ミリ秒) |
| `mLongitude` | int | 経度 (ミリ秒) |
| `mNaturalGuideText` | String | 自然言語ガイドテキスト |
| `mNotificationGuideText` | String | 通知/Android Auto表示用テキスト |
| `mIsCacheInvalidation` | boolean | true=キャッシュ無効化 (動的コンテンツ) |
| `mAdvertiser` | String | 広告主 (スポンサー案内時) |
| `mIsSpotDescriptionTiming` | boolean | スポット説明タイミングフラグ |

### フレーズ組み立ての基本ルール

1つの `PhraseData` 内の `onlineTtsPhraseData[]` が **1つの案内文** を構成する。
配列の各要素が1つのフレーズ断片で、**インデックス順に連結再生** される。

```
onlineTtsPhraseData[0] → "およそ300m先"    (距離)
onlineTtsPhraseData[1] → "この信号を"      (対象物)
onlineTtsPhraseData[2] → "右方向です。"    (方向)

→ 「およそ300m先、この信号を、右方向です。」
```

複数の `PhraseData` が返された場合、連続案内になる:

```
PhraseData[0]: "およそ300m先" + "この信号を" + "右方向、"
PhraseData[1]: "その先" + "分岐です。"

→ 「およそ300m先、この信号を、右方向、その先、分岐です。」
```

### Java側の後処理 (`NTPlayMediaGuideTts.m()`)

ネイティブから返されたフレーズに対し、以下の差し替え処理が入る:

| 元フレーズ | 差し替え先 | 条件 |
|---|---|---|
| `目的地付近です。` | 目的地タイプに応じた文言 | Nc.k マッピング |
| `経由地付近です。` | 実際の経由地スポット名 | ViaSpotList参照 |
| `%spot_description_phrase%` | 動的スポット説明文 | スポット案内タイミング |
| `音声案内を終了します。` | `お疲れ様でした。` | 到着時のキー差し替え |
| `音声案内を中断します。` | (削除) | BG中断時のキー差し替え |

#### 目的地タイプ別の差し替え (`Nc.k.a()`)

| タイプ値 | 差し替え後フレーズ |
|---|---|
| 1 | `目的地付近です。` (そのまま) |
| 2 | `駅入口付近です。` |
| 3 | `バス停付近です。` |
| 4 | `空港付近です。` |
| 5 | `フェリー乗り場付近です。` |
| 6 | `経由地付近です。` |

### TTSキャッシュの解決順序

`com.navitime.components.texttospeech.NTTtsCache.readData()` は以下の順で音声データを探す:

1. カスタムデータ (ユーザーインストール)
2. ローカルファイルデータ
3. 効果音 (SEタイプ)
4. ヒープキャッシュ (インメモリLRU)
5. 動的キャッシュ: `ttsdynamic.db` の `tts_cache_t`
6. 静的キャッシュ: `ttsstatic.db` の `tts_cache_t` ← **768件のプリセット**

検索キーは `(engine, speaker, type, text, volume, speed, pitch, depth, latitude, longitude)` の複合ユニーク制約。

### プリキャッシュ機構

ナビ開始時およびガイドポイント変更時 (`a.z()`)、`referenceGuidanceVoice()` で先読み対象フレーズを取得し、`com.navitime.components.texttospeech.f.j()` で非同期にTTS合成→キャッシュに格納する。動的コンテンツ (`cacheInvalidation=true`) の場合は `CountDownLatch` で最大10秒同期待ちする。

# 01. Project Overview

## App Name

**OneNavi** — "一つのナビで全部済む" をコンセプトとした Android カーナビアプリ。

## Motivation

開発者は以下の 2 つのナビアプリを併用しており、それぞれに不満を抱えている。

### NAVITIME（ドライブサポーター / ツーリングサポーター）

**良い点:**
- 音声案内の品質が極めて高い（信号機案内「2つ目の信号を左です」、通称使用「環八」、自然な日本語「この信号を右折です」）
- ルート選択が日本の交通事情にマッチしている
- アプリ版は高速パネル表示に完全対応（SA/PA/IC/JCT の一覧表示、一般道でも右左折リスト表示）

**悪い点:**
- 自動車用（ドライブサポーター）と二輪車用（ツーリングサポーター）で別アプリ・別契約が必要
- Android Auto 対応が不十分:
  - 経路全体が見えない
  - 経由地の追加ができない
  - 不具合が多い
  - Auto 実行中はスマホ側でアプリを開けない（致命的）
  - Auto 上での検索や他アプリからの intent 共有ができない
- UI/UX の体験が悪い

### Google Maps

**良い点:**
- 美しい UI、スムーズなアニメーション
- ルート案内中でも別経路提案を表示
- 最低限の情報で運転中の視認性が高い
- Android Auto に完璧に対応（Auto 実行中もアプリが開ける、機能差異なし）

**悪い点:**
- 音声案内の質が悪い:
  - 道路名が正式名称すぎる（「環八」→「東京都道311号環状八号線」）
  - 信号機案内がない
  - 日本語として不自然（「信号で右です」）
- 日本特化でないための問題:
  - 交通規制情報が不完全（時間帯制限など）
  - VICS ではなく独自の渋滞情報
  - 交差点拡大図がない
  - 車線案内が不十分（改善傾向）
  - 狭い道・入り組んだ道をルートに採用しがち
- 高速パネル表示に非対応（二つ先の右左折までしか表示しない）

### 両者共通の不満

- 高速道路のパネル表示が不完全（NAVITIME は Auto 版で 3 つまで、Google Maps は非対応）
- 開発者は高速パネルを必須機能と考えている（SA/PA 確認、IC/JCT の目印、通過時刻・距離からの燃費計算）

## Project Goals

1. Google Maps レベルの UI/UX（美しさ・視認性・アニメーション）
2. NAVITIME 以上の音声案内品質（通称辞書、信号機案内、自然な日本語）
3. 高速パネル表示の完全対応（IC/JCT/SA/PA 一覧 + 通過時刻・距離）
4. Android Auto 完全対応（スマホ側アプリも同時操作可能）
5. 自動車 / 二輪車のプロファイル切替
6. OSS として公開（収益化なし、各自 API キー設定前提）

## Usage Flow

1. Google Maps 等の外部マップアプリから **intent share** で OneNavi を起動
2. 共有された座標/地名を目的地としてルート案内を開始
3. ナビ中に経由地追加などのために**最低限の検索機能**を持つ
4. 地点の詳細情報（口コミ・評価など）は**扱わない** — 経路案内に特化

## Non-Goals

- 地点の口コミ・評価・写真表示（Google Maps の役割）
- 徒歩・公共交通ナビ（車両ナビ専用）
- 収益化（広告・課金なし）
- 自前での地図データ・ルーティングエンジン構築（既存サービスを活用）
